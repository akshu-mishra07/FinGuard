Anti-Money Laundering (AML) & Transaction Fraud Graph Analyzer

1. Executive Summary

FinGuard is a state-of-the-art transaction screening and structural money laundering detection platform. Standard rule-based triggers flag basic transaction limits but remain blind to complex multi-account layering networks (mule loops) where money is funneled through intermediaries to hide illicit origins.

The platform addresses this capability gap by combining concurrent transaction buffering with an AI ensemble of deep learning neural networks (PyTorch Autoencoders) and statistical partitioning (Isolation Forests), coupled with an in-memory graph analyzer for directed cycle paths.

During E2E pipeline verification tests, the system successfully isolated a 4-hop circular money laundering loop (ACC_MULE_002 → ACC_MULE_007 → ACC_MULE_003 → ACC_MULE_005 → ACC_MULE_002) in real-time, instantly escalating the transfer risk score to 100% and caching the anomalous profiles in MongoDB.

2. Architecture & Technology Stack

FinGuard is designed for modularity, scalability, and sub-millisecond screening throughput. The backend is built using FastAPI, running Python's asynchronous tasks, and storing dynamic cache matrices in MongoDB. The frontend compliance dashboard runs on Next.js 16.

3. AI Screening Layer

When a transaction is screened, the system builds an 8-dimensional feature vector: [amount, velocity_1m, velocity_10m, freq_1m, freq_10m, out_degree, in_degree, loop_involvement].

PyTorch Spatial Autoencoder

The PyTorch neural network maps the feature vector to a lower-dimensional bottleneck latent space, then reconstructs it. The reconstruction loss is evaluated using Mean Squared Error (MSE):

\(L_{\text{MSE}}(x, \hat{x}) = \frac{1}{d} \sum_{i=1}^{d} (x_i - \hat{x}_i)^2\)

A high reconstruction loss indicates that the transaction features do not match normal behavioral distributions, indicating a spatial statistical vector shift.

Isolation Forest Integration

In parallel, the scikit-learn Isolation Forest evaluates the transaction. The decision function calculates isolation path lengths: anomalies are isolated near the root of the trees.

The final risk score combines both models: $$\text{Risk Score} = \max(\text{Autoencoder Risk}, \text{Isolation Forest Risk})$$

4. Graph Topology & Loop Isolation

A primary laundering tactic (layering) routes funds in circular chains to conceal source funds. FinGuard maintains a directed graph of transactions in a rolling 10-minute window. When a transfer from $S \rightarrow R$ occurs, the Graph Analyzer runs a Depth-First Search (DFS) to see if there is already a path from $R \rightarrow S$: 

def dfs(node, target):
    if node == target:
        path.append(node)
        return True
    if node in visited:
        return False
    visited.add(node)
    path.append(node)
    for neighbor in adj_list[node]:
        if dfs(neighbor, target):
            return True
    path.pop()
    return False

If a path exists, adding the transfer completes the loop. The system flags the loop, extracts the path, and signals the Next.js client via WebSocket to draw glowing red vectors and cashflow path animations.
